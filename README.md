# AGV Web Gateway (Web 3D 可视化网关)

## 📖 项目简介

`agv_web_gateway` 是专为 AGV（自动导引车）Web 端 3D 可视化系统设计的高性能 WebSocket 代理网关。
它的主要职责是作为 **浏览器前端** 与 **AGV 上位机后端服务 (agv_server_pubsub)** 之间的通信桥梁，负责将底层高频、大吞吐量的点云、多边形轮廓、小车位姿等数据流，通过 WebSocket 实时、稳定地推送给前端浏览器，并在内网环境下提供前端静态资源（HTML/JS/CSS）和大型 3D 地图文件（`.pcd`）的 HTTP 下载服务。

## ✨ 核心特性

- 🚀 **高频点云数据透传**：支持小车稀疏点云、避障点云、多边形轮廓、实时定位数据等多种高频数据流的订阅与下发。
- ⚡ **大型 3D 地图直传**：内建 HTTP 服务器，支持直接从 AGV 本地物理磁盘（如 `/home/byd/data/map/...`）以纯二进制流的形式读取并传输数十 MB 的 `.pcd` 点云地图文件。
- ⏱️ **定制化网络握手**：针对 AGV 弱 Wi-Fi 环境下的巨型文件传输场景，深度定制了 WebSocket++ 底层配置，将握手超时时间延长至 60 秒，彻底杜绝传输中断。
- 🔋 **智能省电与带宽控制**：网关在周期性心跳 (`HEART_BEAT`) 中会自动向底层汇报当前活跃的 Web 连接数。当没有浏览器用户查看时，底层自动切断点云订阅，极大节省 AGV 车载主机的 CPU 与网络带宽。
- 🌐 **前端离线化支持**：支持拦截对第三方 JS 库的请求并重定向至本地缓存，确保 AGV 在无外网环境下依然能完美加载 3D 渲染引擎。

## 🛠️ 技术栈

- **语言标准**: C++17 / CMake
- **网络通信**: `Asio` (Standalone, 无 Boost 依赖)
- **WebSocket 协议**: `WebSocket++`
- **日志与数据处理**: `fmt`, `nlohmann/json`

## 🏗️ 架构与工作流

```text
[ Browser (Web 3D) ]
       │  ▲
       │  │ (WebSocket: 指令下发 / 点云JSON回传)
       │  │ (HTTP: 加载 index.html / pcd 地图)
       ▼  │
[ agv_web_gateway ] (127.0.0.1 代理)
       │  ▲
       │  │ (TCP Socket: 二进制私有协议 / 活跃客户端心跳保活)
       ▼  │
[ agv_server_pubsub ] (底层数据服务)
